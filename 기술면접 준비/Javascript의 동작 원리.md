  자바스크립트의 내부 동작 구조는 Meomory heap, Call stack, Callback queue로 이루어져 있다.
  
  heap은 코드 상 변수와 값이 모두 바인딩된 값들을 저장한 저장소와 비슷하다. 또한 바인딩된 결과들의 값들이 올라가거나 내려가는 모든 현상을 저장한다.
  
  stack은 코드 상의 결과를 계산하여 출력하는 공간이다. stack은 js 구조에서 단 하나만 존재하는데 이를 통해 자바스크립트는 싱글쓰레드 기반의 언어라는 것을 확실히 증명할 수 있게 된다.
  
  queue는 Stack에서의 코드 작업 도중 잠시 queue에 보관하여 두는 보관소와 비슷하다. 대표적으로 setTimeOut, Ajax, EventListner Web API 요소들이 queue에 임시로 보관한다.
  만일 stact에서 모든 작업이 종료될 경우 자료구소 queue와 동일하게 FIFO 방식으로 먼저 보관된 순서에 따라 Stack에 계산되어 출력된다.
  
  Stack에서 빈 상태가 되면, Queue의 첫번째 콜백 작업을 Call Stack으로 밀어 넣어 주는 이 과정을 Event Loop라고 부른다.
  
  Event Loop를 쉽게 요약하자면 태스크가 들어오길 기다렸다가 태스크가 들어오면(queue) 이를 처리하고,(stack으로 전달) 처리할 태스크가 없는 경우엔 잠드는, 끊임없이 돌아가는 자바스크립트 내 루프이다.
  
  queue 작업을 깊게 생각하면 이는 결국 비동기적인 작업이 흐르고 있다고 해석하면 좋다.
