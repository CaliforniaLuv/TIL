  node.js는 자바스크립트 기반의 런타임 환경에 맞춰진 플랫폼이다.
  덕분에 브라우저 상에서만 쓰이던 자바스크립트는 서버에서도 서버 사이드 개발에 사용할 수 있는 환경을 제공해주어 분야가 더 확장된 언어가 되었다.
  서버에서 자바스크립트를 파싱할 수 있는 이유는 Chrome V8 엔진을 활용하였기 때문이다.
  또한 node.js는 이벤트 기반의 플랫폼이므로 이벤트 루프와 함께 싱글 스레드 모델을 활용한다.
  많은 요청이 들어와도 비동기적인 non-blocking 방식으로 처리가 되어 싱글 스레드가 활용 가능하다.

  싱글 스레드: 하나의 프로세스에서 작업 흐름 단위가 단일적으로 이루어지는 것

  이벤트 루프: 태스크가 들어오길 기다렸다가 태스크가 들어오면 이를 처리하고, 처리할 태스크가 없는 경우엔 잠드는, 끊임없이 돌아가는 자바스크립트 내 루프이다.

  장점: 모든 요청들이 비동기로 처리 됨으로써 데이터를 실시간으로 다루는 애플리케이션이나 싱글페이지 애플리케이션(SPA), 
       입출력이 잦은 애플리케이션을 개발할 때 뛰어난 효율성을 발휘한다.
  단점: 싱글 쓰레드이기 때문에 하나의 작업이 고사양적인 기반으로 운용된다면 서버가 급격히 저하된다.
