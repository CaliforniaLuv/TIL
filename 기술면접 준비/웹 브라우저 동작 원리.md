웹 브라우저의 기본 구조는 사용자 인터페이스 ⇒ 브라우저 엔진 ⇒ 렌더링 엔진 ⇒ 백엔드 UI, 자바스크립트 해석기, 통신 으로 구축되어 있다. 
여기서 자바스크립트 해석기는 구글 V8 엔진에 해당된다.
사용자 인터페이스는 브라우저 상단 메뉴를 제외한 모든 브라우저 화면 상의 영역을 의미한다.
브라우저 엔진은 사용자 인터페이스와 렌더링 엔진 사이의 동작 제어를 한다.
렌더링 엔진은 요청된 HTML, CSS, JS 파일들을 파싱(해석)하여 화면상으로 표시한다.
렌더링 엔진의 동작 원리를 깊게 설명하자면,

1. html 문서 파일을 파싱하여 DOM 트리를 구축한다.
2. 그 다음 스타일 요소가 있는 외부 css 파일 파싱하여 CSSOM 트리를 구축한다. (CSSOM 트리는 DOM이 화면에 어떻게 스타일 요소로 입혀질지 잡아준다.)
3. 구축된 DOM 트리와 CSSOM 트리를 렌더링 엔진을 통해 렌더 트리로 생성한다.
4. 생생된 렌더 트리는 브라우저 화면 상 배치할 곳 결정한다.
