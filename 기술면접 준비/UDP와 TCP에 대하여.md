 UDP와 TCP는 OSI 7계층에서 전송 계층에 해당된다.
 
 TCP
 3 way handshake를 사용하는 연결지향형 프로토콜이다. 
 패킷이 순서대로 도착되지 않을 경우 다시 서버 단에서 재요청이 가능하므로 순서가 보장된다. (수신이 가능)
 데이터 전송이 성공적으로 이루어진다면 이에 대한 응답을 돌려주므로 데이터 전달 보증이 있다.
 
 UDP
 아무런 기능이 없는 하얀 도화지와 같은 프로토콜이다.
 비연결지향형이므로 3 way handshake 방식을 사용하지 않아, 비교적 데이터 전송 속도가 빠르다.
 그러나 데이터 순서 보장과 데이터 전달 보증이 존재하지 않는다.
 
 HTTP3 부터는 여러 기능이 기본적으로 구성된 TCP와 달리 원하는 방식으로 커스터마이징이 가능한 UDP가 더 유용해졌다.
